# QuickEx API â€” Error Contract

Every non-2xx response from the QuickEx backend follows a single JSON shape.
Clients can rely on this contract for all key flows.

## Response Shape

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable description",
    "request_id": "uuid-v4",
    "fields": { "fieldName": ["validation message"] },
    "details": {}
  }
}
```

| Field | Type | Always present | Description |
|---|---|---|---|
| `success` | `false` | âś… | Always `false` for error responses |
| `error.code` | `string` | âś… | Machine-readable error code (see table below) |
| `error.message` | `string \| string[]` | âś… | Human-readable description |
| `error.request_id` | `string` | When available | UUID that ties the response to server logs. Also sent as `x-request-id` and `x-correlation-id` response headers. Pass this to support when reporting issues. |
| `error.correlationId` | `string` | When available | Alias for `request_id` (kept for backward compatibility) |
| `error.fields` | `object` | Validation errors only | Map of field name â†’ array of messages |
| `error.details` | `object` | Non-production only | Extra context (omitted in production) |

### request_id propagation

The `request_id` is generated by the server on every request (UUID v4) unless
the client supplies its own via the `x-request-id` or `x-correlation-id`
request header. The resolved value is echoed back in:

- `x-request-id` response header
- `x-correlation-id` response header (legacy alias)
- `error.request_id` JSON field
- All server-side log entries for that request

## Error Codes

### Generic / Infrastructure

| Code | HTTP Status | Description |
|---|---|---|
| `INTERNAL_SERVER_ERROR` | 500 | Unexpected server error |
| `RATE_LIMIT_EXCEEDED` | 429 | Too many requests. Check `details.retryAfterSeconds` |
| `VALIDATION_ERROR` | 400 | Request body / query failed validation. Check `fields` |
| `UNAUTHORIZED` | 401 | Missing or invalid API key |
| `FORBIDDEN` | 403 | Valid credentials but insufficient scope |
| `NOT_FOUND` | 404 | Resource does not exist |

### Username

| Code | HTTP Status | Description |
|---|---|---|
| `USERNAME_TAKEN` | 409 | The requested username is already registered |
| `USERNAME_INVALID` | 400 | Username does not meet format requirements |
| `USERNAME_NOT_FOUND` | 404 | No user with that username exists |
| `USERNAME_RESERVED` | 409 | Username is reserved and cannot be claimed |

### Payment Links

| Code | HTTP Status | Description |
|---|---|---|
| `LINK_NOT_FOUND` | 404 | Payment link does not exist |
| `LINK_EXPIRED` | 410 | Payment link has passed its expiry date |
| `LINK_ALREADY_PAID` | 409 | Payment link has already been fulfilled |
| `LINK_CANCELLED` | 410 | Payment link was cancelled by the creator |
| `LINK_LIMIT_EXCEEDED` | 422 | User has reached the maximum number of active links |

### Stellar / Transactions

| Code | HTTP Status | Description |
|---|---|---|
| `STELLAR_INVALID_ADDRESS` | 400 | Public key is not a valid Stellar address |
| `STELLAR_INVALID_ASSET` | 400 | Asset code or issuer is invalid |
| `STELLAR_INVALID_AMOUNT` | 400 | Amount is out of range or has too many decimal places |
| `STELLAR_INVALID_MEMO` | 400 | Memo exceeds length or contains invalid characters |
| `STELLAR_NETWORK_ERROR` | 502 | Horizon API returned an error or is unreachable |
| `STELLAR_TRANSACTION_FAILED` | 422 | Transaction was submitted but rejected by the network |
| `STELLAR_INSUFFICIENT_FUNDS` | 422 | Source account does not have enough balance |
| `STELLAR_PATH_NOT_FOUND` | 404 | No conversion path found between the requested assets |

### Marketplace

| Code | HTTP Status | Description |
|---|---|---|
| `LISTING_NOT_FOUND` | 404 | Marketplace listing does not exist |
| `BID_NOT_FOUND` | 404 | Bid does not exist |
| `LISTING_ALREADY_ACTIVE` | 409 | Username is already listed for sale |
| `BID_BELOW_MINIMUM` | 422 | Bid amount is below the listing minimum |

### Notifications / Webhooks

| Code | HTTP Status | Description |
|---|---|---|
| `WEBHOOK_NOT_FOUND` | 404 | Webhook endpoint does not exist |
| `WEBHOOK_DELIVERY_FAILED` | 502 | Webhook could not be delivered after retries |

## Example Responses

### Validation error (400)

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validation failed",
    "request_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "fields": {
      "amount": ["amount must be a positive number"],
      "asset": ["asset must be a valid Stellar asset code"]
    }
  }
}
```

### Rate limit (429)

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Too many requests. Retry after 30 seconds.",
    "request_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "details": {
      "retryAfterSeconds": 30
    }
  }
}
```

### Business error (404)

```json
{
  "success": false,
  "error": {
    "code": "USERNAME_NOT_FOUND",
    "message": "No user with username 'alice' exists",
    "request_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6"
  }
}
```
